# Token Optimization & System Redesign Plan

## 🎯 Core Goal
Reduce Groq API token usage by **60–80%** per user by intercepting non-essential requests before they reach the LLM, caching repetitive queries, stripping down the system prompt, and eliminating unnecessary chat history.

---

## 8. Token Usage Report (Projection)

### Before Optimization (Current State)
*   **System Prompt:** ~1,200 tokens
*   **User Input:** ~50 tokens
*   **History Payload:** ~400 tokens (grows per turn)
*   **AI Output:** ~300 tokens
*   **Total per Request:** **~1,950 tokens**
*   **Daily Usage (7 Qs):** ~13,650 tokens 
*   **Monthly for 1,000 Users:** **409.5 Million tokens**

### After Optimization (Target State)
*   **Pre-AI Filter (Greetings/Cache):** 0 tokens (handled by Node.js)
*   **System Prompt:** ~350 tokens (Strict, compressed)
*   **User Input:** ~50 tokens
*   **History Payload:** 0 tokens (Default) / ~150 tokens (Only on follow-ups)
*   **AI Output:** ~100 tokens (Compact JSON/Tables)
*   **Total per Request:** **~500 tokens**
*   **Daily Usage (7 Qs):** ~3,500 tokens (assuming 1-2 are cached/greetings)
*   **Monthly for 1,000 Users:** **105 Million tokens**

### 📉 Projected Reduction: **74% Decrease in Token Usage**

## Proposed Changes

### 1. System Prompt Optimization (Compressed)
**Target File:** `server/services/groqService.js`
We will replace the current massive prompt with this highly optimized array:
```javascript
const NUTRITION_PROMPT_COMPRESSED = `Role: Foodity AI. Strict domain: Food, nutrition, health, workouts.
Context: User goal: {goal}, Weight: {weight}, Local Time: {time}.
Rules:
1. Reject non-health topics instantly: "I only assist with nutrition and fitness."
2. Output short, scannable data. NEVER write long paragraphs.
3. If single food provided, ask: "Are you logging this?"
4. If multiple foods provided, return EXACT markdown table: |Food|Kcal|P|C|F|Fib|
5. Rely strictly on provided context: {nutritionData}. Do not hallucinate.`;
```

### 2. "No History" Mode & 5. Intent Filter
**Target File:** `server/controllers/chatController.js`
- **Default:** History array is empty.
- **Trigger Words:** If user input matches `/^(yes|more|continue|add|why|how)/i`, we inject ONLY the last 1 User + 1 Assistant message (not 10).
- **Simple Intent RegEx:** Catch simple agreement/disagreement (`yes`, `no`, `ok`, `thanks`) and return static Node.js responses (0 tokens).

### 3. Response Optimization
**Target File:** `server/services/groqService.js`
- Set `max_tokens: 250` (down from 1024).
- Set `temperature: 0.2` (reduces creative rambling).

### 4. Greeting & Small Talk Removal (Zero Token Strategy)
**Target File:** `server/data/greetings.json` & `server/services/greetingHandler.js`
- We will generate a **1,000+ line JSON dictionary** of greetings using a Node.js script. It will cover permutations of (hi, hello, yo, sup) + (man, dude, friend) + (morning, evening) + regional slang + festival wishes.
- **Logic:** `if (greetingsSet.has(userMessage.toLowerCase().replace(/[^a-z]/g, ''))) return staticGreeting;` -> **0 LLM Tokens.**

### 6. Caching System
**Target File:** `server/controllers/chatController.js` & `server/services/nutritionIntelligence.js`
- We will utilize the absolute Cache Key generated by `parseFoodItems()` in the Nutrition Intelligence layer.
- **New Flow:** If `isFoodQuery` is true and we have a high-confidence USDA/Model hit, we save the completed HTML/Table output in a persistent Database Cache table (`query_cache`). 
- Next time ANY user asks "bingo chips nutrition", they get the cached text instantly -> **0 LLM Tokens.**

---

## 7. Final Architecture Flow
1. **Frontend Request** arrives at `/api/chat`.
2. **Greeting Handler:** Checks 1000+ dataset. (If hit → Return instantly).
3. **Intent Filter:** Checks for "ok", "thanks", "yes". (If handled → Return instantly).
4. **Cache Check:** Normalizes "apple calories" to "1xapple". Looks up in DB. (If hit → Return instantly).
5. **Conditional History:** Checks if query requires context ("yes", "more"). Grabs last 2 msgs only.
6. **AI Call (Groq):** Fires with 350-token prompt + 250 output limit.
7. **Response Cache & Return.**
